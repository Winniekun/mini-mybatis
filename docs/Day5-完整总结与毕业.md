## 🔑 核心知识点汇总

### 1. 执行流程（必须记住）⭐⭐⭐⭐⭐

```
用户调用
  ↓
MapperProxy拦截（动态代理）
  ↓
构建statementId
  ↓
SqlSession处理
  ↓
Executor协调
  ├─> 获取MappedStatement
  ├─> 创建StatementHandler
  └─> 协调执行
  ↓
StatementHandler处理
  ├─> prepare(): #{} → ?
  ├─> parameterize(): 设置参数
  └─> query(): 执行SQL
  ↓
ResultSetHandler映射
  ├─> 遍历ResultSet
  ├─> 创建对象
  ├─> 映射字段
  └─> 类型转换
  ↓
返回结果
```

---

### 2. 四大核心对象（必须理解）⭐⭐⭐⭐⭐

```
1. Executor（执行器）
   - 作用：协调SQL执行
   - 类型：Simple、Reuse、Batch
   - 职责：管理流程，不直接操作JDBC

2. StatementHandler（语句处理器）
   - 作用：封装JDBC操作
   - 方法：prepare、parameterize、query/update
   - 职责：创建Statement、设置参数、执行SQL

3. ParameterHandler（参数处理器）
   - 作用：处理SQL参数
   - 职责：Java对象 → JDBC参数

4. ResultSetHandler（结果处理器）
   - 作用：处理结果集
   - 职责：ResultSet → Java对象
   - 能力：自动映射、类型转换
```

---

### 3. 面试高频题（必须会答）⭐⭐⭐⭐⭐

```
Q1: #{} 和 ${} 的区别？
A: #{}预编译防注入，${}字符串替换有风险

Q2: MyBatis的执行流程？
A: MapperProxy → SqlSession → Executor → StatementHandler → ResultSetHandler

Q3: 一级缓存和二级缓存的区别？
A: 一级SqlSession级别默认开启，二级Mapper级别需配置

Q4: 如何防止SQL注入？
A: 使用#{}，底层用PreparedStatement，参数会被转义

Q5: MyBatis的四大核心对象？
A: Executor、StatementHandler、ParameterHandler、ResultSetHandler
```
